# DesignStudio CRM — ДЗ-9 (Lite)

Локальная CRM для отдела продаж: **Next.js 14 + Supabase (Docker) + RLS**.

**Путь:** `Projects/ДЗ-9/designstudio-crm/`

---

## Быстрый старт

### 1. Docker Desktop

Запустите Docker Desktop (иконка кита в трее). Без него `supabase start` не работает.

### 2. Supabase

```powershell
cd Projects\ДЗ-9\designstudio-crm
npx supabase start
```

Первый запуск ~3–5 мин. Сохраните ключи:

```powershell
npx supabase status
```

### 3. Переменные окружения

Скопируйте `.env.local.example` → `.env.local` и вставьте **anon key** и **service_role key** из `supabase status`.

### 4. Next.js

```powershell
npm install
npm run dev
```

Откройте http://localhost:3000

## Регистрация пользователей (ДЗ)

| Роль | Email | Пароль (пример) |
|------|-------|-----------------|
| Менеджер | manager@designstudio.ru | Test1234! |
| Админ | admin@designstudio.ru | Admin1234! |

Регистрация через форму на `/login` → «Нет аккаунта — регистрация».

Подтверждение email **отключено** в локальном Supabase (`enable_confirmations = false`).

---

## SQL после регистрации

В **Supabase Studio** http://localhost:54323 → SQL Editor выполните скрипт:

`supabase/seed_after_signup.sql`

Он:
- назначит роль `admin` для admin@designstudio.ru (+ `app_metadata` для RLS);
- привяжет двух лидов менеджеру.

**Важно:** после смены `app_metadata` админ должен **выйти и войти снова**.

## Терминал (важно)

- **Один** терминал VS Code в папке проекта — `npm run dev`
- Порт в терминале = порт в браузере (`Local: http://localhost:3000`)
- Не запускать `npm run dev` в двух окнах — иначе 3000/3001 и 404
- `designstudio-crm` — имя папки, **не команда**; нужны `npm run dev`, `npx supabase start`
